Method 1 (Just 4 steps)

Simplest solution without using angular-cli-ghpages tool or any external tool.
Step 1
Create Github Repository :

Assuming you already have an account on github, create a new repo. Choose your github repo name wisely as it is going to be used to url used to access the webapp in the browser when its live irrespective of your angular project name. The url that gets formed has format : https://[username].github.io/[reponame]/ ( My Website : https://saurabhrathi.github.io/heroes)
The Repository should be marked public.

Step 2
Build your Angular App :

We generally use ng build --prod command for an angualr app to generate the static files (all the typescript code gets converted into js files and overall code gets optimized and minified) which are ready to be deployed on a server that can serve static files (html, css, js). By default, this deployable static files are generated in the /dist/{project-name}
But here we need to do some changes. We want to generate files in "docs" folder. Because github explicitly gives you option for this folder which you will see later in this tutorial. So, we need to make a small change in the angular.json file and change the outputpath value to “docs/”. Now use the command : ng build --prod --baseHref=” https://username.github.io/respository-name/”
Note that the username and repository name will be your GitHub username and repository name. Example : ng build --prod --base-href "https://SaurabhRathi.github.io/heroes/".

Step 3
Push code to remote repo in github

Use commands :
git remote set origin {your GitHub repository URL} (Need to do this only once).
git add .
git commit -m "generated deployable code in docs folder"
git push origin master (You can use any remote branch, i'm using master branch for this tutorial).

Step 4
Repo settings for github page

Open your github repository in a browser and go to settings. Scroll down to github pages section.
Here it asks for the "Source" of your deployable code (static code). You can set it by selecting the branch and folder to master branch and docs folder. Save the changes.

Once this is done, you will see a message that your Website is live on this url : https://username.github.io/respository-name/” (Example : https://SaurabhRathi.github.io/heroes/)
